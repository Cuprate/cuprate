# cuprated 0.0.9 Molybdenite (2026-04-15)
Cuprate is a modular Monero node written in Rust. To get started, see: <https://user.cuprate.org>.

**This release contains an upgrade to the database; sync speed is multiples faster and disk usage is now below [`monerod`](https://github.com/monero-project/monero) (~202GB vs ~292GB).**

The database format has changed from [LMDB](http://www.lmdb.tech) to a hybrid system using [Fjall](https://github.com/fjall-rs/fjall) and [Tapes](https://github.com/Cuprate/Tapes); previous database files are incompatible, please delete previous [Cuprate files](https://user.cuprate.org/resources/disk.html) and re-sync.

## Changes
- Generic SOCKS5 proxy support, clear-net over daemon ([#549](https://github.com/Cuprate/cuprate/pull/549))
- Delay Tor P2P zone start until after sync ([#576](https://github.com/Cuprate/cuprate/pull/576))
- Broadcast newly added alt blocks to peers ([#577](https://github.com/Cuprate/cuprate/pull/577))
- Add auto option for Tor ([#578](https://github.com/Cuprate/cuprate/pull/578))
- Make Arti an optional feature flag ([#582](https://github.com/Cuprate/cuprate/pull/582))
- Ignore fluffy blocks older than 10 blocks ([#583](https://github.com/Cuprate/cuprate/pull/583))
- Event-driven block synchronization ([#584](https://github.com/Cuprate/cuprate/pull/584))
- LMDB -> Fjall/Tapes ([#587](https://github.com/Cuprate/cuprate/pull/587))
- Use last ring input as youngest output ([#593](https://github.com/Cuprate/cuprate/pull/593))
- Set tx to public after promotion from stem state ([#594](https://github.com/Cuprate/cuprate/pull/594))
